Locacoeur Cloud API 1.0.2
  • application/json

The API is based on the MQTT protocol and this document specifies the MQTT command requests, responses and events to/from the Locacoeur cloud server.

Features

The API features are:

  • Downstream commands for the phone numbers for the call center support.
  • Every command has a corresponding upstream result message with the result of the operation
  • Realtime upstream events with timestamp for alert triggering, status notifications, GPS location and etc.
  • The commands have also subtopic 'get', which allows to request the current value of the command parameter (if any), e.g. LC1/{serial}/command/config/get will ask for the current configuration, rather than setting new device configuration.

Initial device configuration

After the initial power-up of the device, it will use the integrated 4G LTE modem to automatically connect to the cloud MQTT broker. The default MQTT network address and credentials are programmed in the device when it is manufactured in the factory. The device will need a 4G LTE SIM card to connect to the cellular network. On the first connection the device will do the following:

  • Send the device status
  • Wait for 10seconds (configurable) to receive any MQTT commands or configuration

LEDs

  • 🔌 LED 1 - Power Status
    • Green: Device is powered by an external power supply.
    • Red: Device is running on battery power.
  • ⚡ LED 2 - Defibrillator Status
    • Green: Defibrillator is functioning normally.
    • Red: Indicates a fault in the defibrillator if:
      • The green indicator LED on the defibrillator is not lit, or
      • No beep/sound has been detected in the last 1 hour.
  • 📊 LED 3 - Monitoring Mode
    • Green: Monitoring is active (e.g., remote help, alert support) is enabled.
    • Red: Monitoring is disabled.
  • 🧑‍⚕️ LED 4 - Assistance Mode
    • Green: Assistance mode (e.g., remote help, alert support) is enabled.
    • Red: Assistance is disabled.
  • 🌐 LED 5 - MQTT Connectivity
    • Green: Connected to the MQTT broker successfully.
    • Red: Disconnected from the MQTT broker or cannot establish a connection.
  • 🌡️ LED 6 - Environmental Control (Fan/Heater & Temperature Status)
    • Green: Fan or heater are currently operating.
    • Off: Neither the fan nor the heater is active (normal state if temperature is in range).
    • Red: Temperature is outside safe limits:
      • Below 5°C or
      • Above 40°C.

  • REQUEST LC1/{serial}/command/update

    Firmware update request downstream topic.
    The request in this topic indicates that there's a new version of a specific OTA package, and the link of an HTTP repository where the device can download the new version from.
    The device manages itself the download and the full firmware update process including the version check, integrity check and the signature validation.
    The device will return in the response if the command was successfully received and the update has been initiated.
    If there was a failure during the OTA update process setup, an error will be returned in the response message.

    defibrillator
    required
    integer

    The defibrillator health status. Healthy=0. Negative number on error.
    The following error codes are returned:

    • 0 = Defibrillator is ready for use. All self-tests passed.
    • -1 = Optical sensor fault: Sensor detects a red indicator — defibrillator not ready.
    • -2 = Battery failure: Defibrillator does not emit a beep signal — battery likely dead.
    • -3 = Microphone malfunction: No input detected from the microphone component.
    • -4 = Current sensor failure: Unable to measure current — sensor unresponsive or damaged.
    • -5 = Fan failure: Fan not operating — cooling functionality compromised.
    • -6 = Heater failure: Heating element not functioning — defibrillator temperature control affected.
    • -7 = Temperature sensor fault: No valid temperature data retrieved.
    • -8 = Accelerometer failure: No motion or orientation data received.
    • -9 = Color sensor failure: Unable to detect color levels — sensor may be faulty or disconnected.
